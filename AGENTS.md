# Repository Guidelines

## 定位
本仓库是基于 `Void` 初始化的具体项目仓库。
`Void` 定义项目宪法与目录契约；`.void/` 承载项目生命周期内容。

## Void 契约
- `Void`：定义规则、边界与目录语义
- `.void/idea/`：记录动机、目标、问题与约束
- `.void/blueprint/`：记录系统边界与高层结构
- `.void/solution/`：记录方案、决策与取舍
- `.void/implementation/`：记录里程碑、任务、阻塞与下一步
- `.void/validation/`：记录验证过程、证据、结果与结论
- `.void/release/`：记录阶段成果、可用范围与已知限制

## 协作规则
- 先判断内容属于代码、`Void` 还是 `.void/`
- 意图、方案、实施、验证、发布相关内容优先进入 `.void/`
- 不在仓库根目录长期堆放零散讨论、进度流水账或验证记录
- 修改 `Void` 时保持克制，只有契约需要变化时才修改

## Codex 约束
- 将 `Void` 视为本项目最高层契约
- 发现目录语义与文档位置冲突时，优先维护 `Void` 的一致性
- 项目特有约束可在本文件追加，但不应删除以上公共约定

## 项目特有约束
在此追加本项目自己的技术、协作或发布要求。
